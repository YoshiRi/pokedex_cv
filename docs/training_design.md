# Training Phase Design

## Goal

Train a YOLOv8 object detection model that can detect and classify Pokemon in images,
given a dataset of synthetic composites generated by the annotation pipeline.

---

## Architecture Choice

**Baseline: YOLOv8n (nano)**
- Smallest model in the YOLOv8 family — fast iteration, low GPU requirement
- Ultralytics API wraps training, validation, and export uniformly
- Established baseline before trying RT-DETR or larger variants

**Alternative (future): RT-DETR**
- Transformer-based; potentially better on small/unusual-aspect objects like sprites
- Higher GPU cost; skip for PoC

---

## Class ID Design

### National Pokedex IDs are NOT contiguous

Full dataset: 1025 Pokemon with national IDs 1–1025 (some future IDs may be skipped).
YOLO class labels must be contiguous 0-indexed integers.

### `class_map` list

Each experiment config defines `class_map`: an ordered list of national Pokedex IDs.

```
class_map: [1, 4, 7, 25, ...]
```

- Index in the list = YOLO class_id written to label files
- Value at that index = national Pokedex number stored in AnnotationStore
- `data.yaml` `names` list follows the same ordering

This is a **per-experiment** mapping. Full 1025-species training would have
`class_map: [1, 2, 3, ..., 1025]` (i.e., class_id = pokedex_id - 1).

The mapping is applied at **export time** (`annotation/export/to_yolo.py`),
not at annotation time, so annotations always store the true Pokedex ID.

## Dataset Layout (YOLO format)

```
datasets/poc_20species/yolo/
├── data.yaml
├── images/
│   ├── train/   composite_*.jpg  (80%)
│   ├── val/     composite_*.jpg  (10%)
│   └── test/    composite_*.jpg  (10%)
└── labels/
    ├── train/   composite_*.txt
    ├── val/     composite_*.txt
    └── test/    composite_*.txt
```

`data.yaml` contains:
- `path`: absolute path to the dataset root
- `train/val/test`: relative subdirectory paths
- `nc`: number of classes (len of class_map)
- `names`: Pokemon English names in class_map order

## Training Configuration

Hyperparameters live in two places:

1. **`training/configs/yolov8n_poc.yaml`** — Ultralytics override YAML
   - Passed as `cfg=` kwarg to `YOLO.train()`
   - Controls lr, augmentation flags, etc.

2. **`configs/poc_20species.yaml` `training:` section** — high-level experiment params
   - `model`, `epochs`, `imgsz`, `batch`, `patience`, `project`, `name`

### Augmentation

Ultralytics YOLOv8 has built-in augmentation (mosaic, mixup, random flip/scale/hsv).
No custom `dataset/augment.py` is needed for the baseline.

Mosaic is especially beneficial here because composites already place sprites on
varied backgrounds — mosaic further mixes backgrounds and improves robustness.

---

## Evaluation Strategy

Two evaluation modes:

### 1. Synthetic validation (`--mode val`)
- Run `YOLO.val()` on the `val` split of the YOLO dataset
- Reports mAP@50, mAP@50:95, precision, recall per class
- Fast automated check; used during training loop

### 2. Real-image test (`--mode real`)
- Load model and run inference on a directory of real images
- Saves annotated images with predicted bboxes + confidence
- Reports per-class detection counts (no ground truth available at this stage)
- Needed to assess sim-to-real transfer gap

## Training Script

```bash
# Run full pipeline and train
python annotation/pipeline.py --raw-dir data/sprites/pokeapi --output datasets/poc_20species/raw_annotated --composite

python annotation/export/to_yolo.py --config configs/poc_20species.yaml --annotations datasets/poc_20species/raw_annotated/annotations.jsonl --output datasets/poc_20species/yolo

python training/train.py --config configs/poc_20species.yaml

# Evaluate
python training/evaluate.py --config configs/poc_20species.yaml
python training/evaluate.py --config configs/poc_20species.yaml --mode real --images /path/to/real_pokemon_images/

# Export
python training/export.py --config configs/poc_20species.yaml --format onnx
```

## Risks

| Risk | Mitigation |
|---|---|
| Synthetic-only training doesn't generalize | Evaluate on real images; add real data later |
| YOLOv8n too small for 1025 classes | Use YOLOv8m/l for full dataset |
| Sprite composites too unrealistic | Improve background diversity; add real backgrounds |
| Mosaic merges Pokemon bboxes incorrectly | Monitor val loss; disable if problematic |
